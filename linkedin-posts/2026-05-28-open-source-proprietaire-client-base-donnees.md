# Post LinkedIn

Choisir un client de base de données, c'est aussi choisir un modèle de licence. Et ce choix conditionne tout : qui peut lire le code, qui paie, qui décide de la roadmap, et ce qui se passe le jour où l'éditeur disparaît.

Trois modèles coexistent :

1. Open source pur (DBeaver CE, pgAdmin) : auditabilité totale, coût direct nul, support communautaire.

2. Propriétaire fermé (DataGrip, TablePlus) : UX léchée, support contractuel, mais dépendance complète à l'éditeur et coût qui grossit avec l'équipe.

3. Open core (Beekeeper, QoreDB) : code ouvert sur le chemin critique (vault, drivers, exécution), modules payants sur la valeur ajoutée (fédération, contrats de données, IA).

J'ai écrit un comparatif structuré autour de quatre critères qui comptent vraiment dans la durée : auditabilité, pérennité, support, prix long terme. Avec les chiffres et les cas d'usage concrets.

Pour QoreDB, le choix de la BUSL 1.1 plutôt que d'une licence purement propriétaire est délibéré : code lisible, modifiable pour un usage non concurrentiel, bascule en Apache 2.0 au bout de 4 ans. Pas de bait-and-switch.
