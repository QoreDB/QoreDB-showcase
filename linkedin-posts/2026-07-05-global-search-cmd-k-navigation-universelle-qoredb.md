# Post LinkedIn

Un client de base de données finit toujours par accumuler des surfaces : sidebar des connexions, onglets, Query Library, commandes de menu, modales pour la fédération, les diffs, les snapshots.

Sans point d'entrée unique, chaque action demande de deviner où elle est rangée.

Dans QoreDB, j'ai fait le choix d'une seule combinaison de touches (Cmd+K) qui indexe six sources en local : commandes, features, connexions sauvegardées, requêtes favorites, historique et Query Library.

Pas de fuzzy matching, pas de score de pertinence : tri par intention, sous-chaîne, 15 résultats max. Le comportement est prévisible parce que les règles sont fixes.

Résultat : trois lettres, Enter, on passe d'une connexion staging à un notebook d'exploration en moins de deux secondes. Sans réseau, sans index tiers, sans compte.
